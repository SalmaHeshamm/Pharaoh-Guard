# Lost Person Protocol

## Trigger conditions
- `Emergency_Type == "Lost Person"`
- Higher baseline likelihood at sites with `School_Trips == 1` and high
  `Crowd_Density`.

## Immediate actions (priority 2)
1. Notify all entry/exit gate staff with a description broadcast.
2. Lock down exit gates temporarily only if `Risk_Level == "Critical"` —
   otherwise keep gates open to avoid creating a secondary crowd-crush risk.
3. Use Security_Staff already on-site before requesting reallocation; a Lost
   Person case rarely needs additional headcount, it needs coordination.

## Escalation
- If unresolved after 20 minutes, escalate to Police_Units and issue a
  public announcement at the site's PA system (Karnak, Luxor and Giza have
  the highest event traffic and PA coverage).

## Notes
- School trip hours (typically 09:00–12:00) at Giza Pyramids and Grand
  Egyptian Museum historically show the highest Lost Person frequency —
  proactively increase Security_Staff visibility near main gates during
  these windows rather than waiting for an incident.
